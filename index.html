Hugo + Netlify + Cloudinary

Stupid Simple Static Site Solution
I've had some bench time recently and this is one of the projects I've done. It's the one that wasn't just following a tutorial and it only took a few days to spin up a simple site while learning all systems involved.

How is this useful to Mutual Mobile?

We're generally not in the business of building simple websites, but if we have to put up a couple of marketing pages in support of an app we build, this could be an easy solution.

Static Site Generator?

  • Templates
  • Markdown
  • Generated HTML
I should probably explain what I mean by "Static Site Generator" because it sounds a bit buzzwordy. Generally how they work is you define some templates, in Hugo's case it's a Go templating system, and content, usually in the case of markdown files with frontmatter metadata. The markdown is run through the templates to generate flat HTML files. These are then served to the user from a file server with no database backend.

Why is this a good thing?

Years of Wordpress and Drupal have given me PCMSSD

(Post Content Management System Stress Disorder)
I don't know who's spent much time supporting CMSes like Wordpress or Drupal, but you can probably relate to my pain. Constant maintenance updates of core and plugins, high-severity security updates that need to go into production immediately, ridiculously tech stacks to get them to run at scale. Multiple MySQL database servers, multiple Apache/NGINX webservers, PHP, Varnish HTTP Proxy, CDN in front of Varnish. And just try to get a matching local environment.

Lots of them

Jekyll, Hugo, Hexo, Octopress, Pelican, Brunch, Metalsmith, Wintersmith, Middleman, Harp, DocPad, Sculpin, etc...

There's a lot to choose from. It's a fairly arbitrary choise that boils down to personal preference.

Hugo

  • Static Site Generator
  • Written in Go
  • Installable binary
Hugo is the static site generator I ended up using mostly because it's an installable binary. There are dozens, hundreds of SSGs to choose from. I used Hugo mostly because I wouldn't be tempted to hack it because it's a binary. Compare that to Metalsmith (https://metalsmith.io/) where you string together a system of plugins to hopefully arrive at something. I've gotten started with that before and never actually got around to building the site I was trying to build.

Drawbacks

  • Handling user interaction (form submission)
  • Editing content by non-technical users
As flat HTML files with a very minimal server, there's not much interactivity. Also most of the editing is done in a text editor and then checked into a git repo, so not particularly user friendly unless you're a dev. However Netlify has done a lot of work with their platform to alleviate these problems.

Netlify

Hosting solution for static sites

  • Easy setup
  • Simple deploys
  • SSL via Let's Encrypt
  • Free! (at my scale)
Netlify was by far the easiest hosting setup I've done. All I had to do was give Netlify permission to the Github repo that contained the Hugo site.

Netlify Goodies

  • Form submission handling
  • Lambda functions
  • Identity services
  • Large Media Handling
These are some of the features that Netlify offers to work around the limitations of a static site. The only one I used was the form submission handling. It looks like the large media handling probably made Cloudinary redundant. Take a look at the Netlify admin.

Cloudinary*

Upload a large image and create resized derivatives based on parameters in the url.

*optional
Resizing images is a pain in the ass and Cloudinary takes care of all that. It's another service I've used that just works how I think it should. And at my scale it's free! Take a look at the admin.
