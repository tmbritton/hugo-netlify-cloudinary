Static Site Generator?

  • Templates
  • Markdown
  • Generated HTML
  • Static Site Generator is a bit buzzwordy
  • Write some templates
  • Content, usually Markdown
  • Run the Markdown through the templates to generate HTML

Why is this good?

Years of Wordpress and Drupal have given me PCMSSD

(Post Content Management System Stress Disorder)
  • CMSes like Wordpress or Drupal
  • constant updates of core and plugins
  • high-severity security updates
  • ridiculous tech stacks: Multiple MySQL database servers, multiple Apache/NGINX webservers, Memcache, PHP, Varnish HTTP Proxy, CDN in front of Varnish.

Drawbacks

  • Handling user interaction (form submission)
  • Editing content by non-technical users
  • HTML files with a very minimal server
  • editing is done in a text editor and then checked into a git repo
  • Netlify has done a lot of work with their platform to alleviate these problems

    • Lots of them

    Jekyll, Hugo, Hexo, Octopress, Pelican, Brunch, Metalsmith, Wintersmith, Middleman, Harp, DocPad, Sculpin, etc...

    Arbitrary choise that boils down to personal preference.

    Hugo

    • Static Site Generator
    • Written in Go
    • Installable binary
    Chose Hugo because:
    • it's an installable binary
    • I wouldn't be tempted to hack it because it's a binary
    • Compare that to Metalsmith

    Netlify

    Hosting solution for static sites

    • Easy setup
    • Simple deploys
    • SSL via Let's Encrypt
    • Free! (at my scale)
    • Easiest hosting setup I've done
    • Just connected it to the github repo
    • Added config file to repo telling it what version of hugo to use
    • Firebase hosting may be an alternative
    • Or and S3 bucket if you're into the whole AWS thing

    Netlify Goodies

    • Form submission handling
    • Lambda functions
    • Identity services
    • Large Media Handling
    • The only one I used was the form submission handling
    • large media handling probably made Cloudinary redundant.

    Cloudinary*

    Upload a large image and create resized derivatives based on parameters in the url.

    *optional
    • Resizing images is a pain in the ass
    • Just works

    Bonus!

    Pure.css

    Tiny CSS library that makes things look reasonably good without hijacking the markup.

    • Didn't want to decide what every form element looks like
    • Wanted something smaller than Bootstrap or Foundation
    • By people who made Normalize.css
    • Handles form error states with CSS
